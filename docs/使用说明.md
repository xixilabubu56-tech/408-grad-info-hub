# 408 研招信息库使用说明

## 1. 项目简介

本项目用于查询全国统考 408 相关的研究生招生信息，当前项目由三部分组成：

- 前端：React + Vite，负责网页展示
- 后端：FastAPI，负责提供接口
- 数据脚本：Python + MySQL，负责初始化和更新数据

当前项目默认使用本地 MySQL 数据库，前端通过后端接口读取数据库中的院校、专业、历年数据和通知信息。

---

## 2. 目录说明

```text
information/
├── backend/              后端接口
├── frontend/             前端网页
├── scraper/              数据初始化与更新脚本
├── docs/                 文档
└── README.md             项目总说明
```

最重要的几个文件：

- `backend/.env`：数据库连接配置
- `backend/init_db.py`：初始化数据库表结构
- `scraper/seed_comprehensive_408.py`：写入 40 所院校的完整数据
- `backend/main.py`：后端启动入口

## 4. 第一次使用：完整启动流程

### 4.1 配置 MySQL

先确认 MySQL 已启动：

```bash
mysql.server start
```

然后进入 MySQL：

```bash
mysql -u root -p
```

如果你的 root 没密码，也可以：

```bash
mysql -u root
```

创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS grad_info_408 DEFAULT CHARACTER SET utf8mb4;
```

创建完成后退出：

```sql
exit
```

---

### 4.2 配置后端数据库连接

打开 [backend/.env](file:///Users/macbook/Desktop/408project/information/backend/.env)。

当前默认配置是：

```env
DATABASE_URL=mysql+pymysql://root:@localhost:3306/grad_info_408?charset=utf8mb4
```

如果你的 MySQL 有密码，请改成：

```env
DATABASE_URL=mysql+pymysql://root:你的密码@localhost:3306/grad_info_408?charset=utf8mb4
```

例如：

```env
DATABASE_URL=mysql+pymysql://root:123456@localhost:3306/grad_info_408?charset=utf8mb4
```

---

### 4.3 安装后端依赖

进入后端目录：

## 6. 数据怎么更新

当前项目的数据更新方式分为两类。

### 6.1 重新导入当前完整数据

如果你只是想把数据库重刷成当前项目内置的 40 校数据，执行：

```bash
cd /Users/macbook/Desktop/408project/information/scraper
source venv/bin/activate
python seed_comprehensive_408.py
```

注意：

- 这个脚本会先清理旧数据，再重新写入完整数据
- 适合“重建数据库”或“刷新到当前项目内置数据”的场景

---

### 6.2 后续接入真实爬虫更新

当前项目已经具备数据脚本结构，但“逐校官网自动抓取”还需要继续扩展。

你目前可以这样理解：

- `seed_comprehensive_408.py`：当前稳定可用的数据初始化/刷新脚本
- `fetch_and_save.py`、`fetch_408_details.py`：爬虫实验/早期脚本，不建议直接作为正式更新入口

所以目前正式推荐的更新方式仍然是：

```bash
python seed_comprehensive_408.py
```

如果以后继续扩展自动爬虫，建议新建专门的正式更新脚本，再接入定时任务。

## 7. 怎么和 MySQL 对接

### 7.1 当前对接方式

项目通过 SQLAlchemy + PyMySQL 连接 MySQL。

配置文件在：

- [database.py](file:///Users/macbook/Desktop/408project/information/backend/database.py)
- [backend/.env](file:///Users/macbook/Desktop/408project/information/backend/.env)

连接字符串格式：

```env
DATABASE_URL=mysql+pymysql://用户名:密码@主机:端口/数据库名?charset=utf8mb4
```

例如：

```env
DATABASE_URL=mysql+pymysql://root:123456@localhost:3306/grad_info_408?charset=utf8mb4
```

---

### 7.2 怎么确认已经连上 MySQL

你可以运行：

```bash
cd /Users/macbook/Desktop/408project/information/backend
source venv/bin/activate
python init_db.py
```

如果看到数据库表创建成功，说明连接正常。

你也可以直接登录 MySQL 查看：

```bash
mysql -u root -p
```

然后执行：

```sql
USE grad_info_408;
SHOW TABLES;
SELECT COUNT(*) FROM institutions;
```

---

### 7.3 常用数据库检查语句

## 8. 常见问题

### 8.1 `npm run build` 报 `ENOENT`，找不到 package.json

原因：你在根目录执行了 npm 命令。

错误目录：

```bash
cd /Users/macbook/Desktop/408project/information
npm run build
```

正确目录：

```bash
cd /Users/macbook/Desktop/408project/information/frontend
npm run build
```

因为只有 `frontend/` 目录下才有 `package.json`。

---

### 8.2 前端能打开，但没有数据

先检查三件事：

1. MySQL 是否启动
2. 后端是否启动在 8000 端口
3. 数据是否已经导入

建议顺序：

```bash
mysql.server start
cd /Users/macbook/Desktop/408project/information/backend
source venv/bin/activate
uvicorn main:app --reload --port 8000
```

另开一个终端：

```bash
cd /Users/macbook/Desktop/408project/information/scraper
source venv/bin/activate
python seed_comprehensive_408.py
```

---

### 8.3 8000 端口被占用

可以先查占用进程：

```bash
lsof -i :8000
```

强制结束：

```bash
lsof -ti:8000 | xargs kill -9
```

然后重新启动后端。

---

### 8.4 MySQL 连不上

先确认 MySQL 是否启动：

```bash
mysql.server start
```

再确认配置文件中的用户名、密码、数据库名是否正确：

- [backend/.env](file:///Users/macbook/Desktop/408project/information/backend/.env)

---

### 8.5 页面还是旧数据

可能原因：

- 后端没重启
- 浏览器缓存还在
- 数据脚本没重新执行

建议：

1. 重新运行 `python seed_comprehensive_408.py`
2. 重启后端
3. 刷新浏览器
